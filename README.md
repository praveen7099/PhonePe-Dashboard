# PhonePe-Dashboard
1. Project Title
PhonePe Transaction Analysis Dashboard | Power BI Business Intelligence Project

2. Short Description / Purpose
The PhonePe Transaction Analysis Dashboard is an interactive Power BI report designed to analyze digital payment transactions across multiple dimensions such as transaction volume, transaction value, user demographics, service categories, and payment success rates. The dashboard enables stakeholders to monitor overall business performance, identify user behavior patterns, evaluate payment success, and gain actionable insights that support strategic business decisions.

3. Tech Stack
The project was built using the following technologies:
- Power BI Desktop – Dashboard creation and data visualization
- Power Query – Data cleaning and transformation (ETL)
- DAX (Data Analysis Expressions) – Measures, KPIs, and calculated fields
- Excel / CSV Dataset – Source data
- Power BI Visualizations

4. Data Source
The dashboard is created using a transactional dataset representing PhonePe payment activities.
The dataset contains information such as:
- Transaction ID
- Transaction Date
- Transaction Value
- Payment Status
- User Information
- Service Category
- Age Group
- Month
- Customer Details
Note: This project uses a sample dataset for learning and portfolio purposes and is not affiliated with or sourced from PhonePe's official internal data.

5. Features / Highlights

A. Business Problem
Digital payment companies process millions of transactions daily across various services. Business teams require a centralized reporting solution to answer questions like:
- How many transactions occur every month?
- What is the overall transaction value?
- Which services generate the highest revenue?
- Which customer segments contribute the most?
- What is the payment success rate?
- How do weekday and weekend transactions differ?
- Which users contribute the highest transaction value?
Without an interactive dashboard, identifying these trends becomes time-consuming and inefficient.

B. Goal of the Dashboard
The dashboard aims to:
- Monitor overall payment performance.
- Track transaction volume and transaction value.
- Measure payment success rate.
- Analyze customer demographics.
- Identify top-performing services.
- Compare weekday vs weekend transaction behavior.
- Highlight top-value customers.
- Provide business insights for strategic decision-making.

C. Walkthrough of Key Visuals
🔹 KPI Cards (Top Section)
The dashboard starts with high-level KPIs that provide an instant overview of business performance:
- Total Transactions: 300K
- Total Transaction Value: ₹3.47 Billion
- Unique Users: 108K
- Successful Payment Rate: 96%
These KPIs help management quickly assess platform performance.

🔹 Transactions Over Time
A dual-axis line chart visualizes:
- Monthly Transaction Count
- Monthly Transaction Value
Insights:
- Transaction activity fluctuates across the year.
- Peak transaction periods occur around July and October.
- Consistent transaction value indicates stable business growth.

🔹 Age Segment Contribution
The donut chart categorizes users into:
- Millennials
- Gen X
- Gen Z
- Boomers
Insights:
- Millennials contribute the highest share of transactions.
- Gen X closely follows.
- Gen Z has moderate participation.
- Boomers contribute the least.
This helps marketing teams target customer segments effectively.

🔹 Service Transaction Value Analysis
Horizontal bar chart compares transaction value across services.
Services include:
- Loans
- Insurance
- Money Transfer
- Recharge & Bills
Key Finding:
- Loans generate the highest transaction value (₹2.5B).
- Insurance is the second-largest contributor.
- Recharge & Bill Payments contribute the least.
This enables product teams to prioritize high-value services.

🔹 Top 5 Users
Column chart ranks customers based on transaction value.
The visual identifies the highest-value customers, enabling:
- Customer retention strategies
- Loyalty programs
- VIP customer targeting

🔹 Weekday vs Weekend Usage
A donut chart compares transaction activity between weekdays and weekends.
Observation:
- Approximately 72% of transactions occur on weekdays.
- Around 28% occur on weekends.
This indicates users perform most financial activities during working days.

🔹 Interactive Filters
Users can dynamically filter the report by:
- Month
- Payment Status
- Successful
- Pending
- Failed
These slicers allow detailed exploration without changing the underlying report.

🔹 Insights Panel
The dashboard includes a dedicated business insights section summarizing key observations.
Example insights:
- Weekdays record the highest number of transactions.
- Loans generate the maximum transaction value.
- Payment success rate remains consistently high at 96%.
This provides executives with quick, actionable takeaways.

D. Business Impact & Insights
The dashboard enables decision-makers to:
- Monitor business performance in real time.
- Detect seasonal transaction trends.
- Identify the most profitable service categories.
- Understand customer demographics.
- Improve payment success monitoring.
- Design targeted marketing campaigns.
- Increase customer engagement through demographic insights.
- Optimize resource allocation based on transaction behavior.
- Prioritize high-value services like Loans and Insurance.
Key Business Insights
📈 Over 300K transactions were processed.
💰 Total transaction value reached ₹3.47 Billion.
✅ 96% payment success rate reflects a highly reliable payment system.
👥 Millennials represent the largest customer segment.
🏦 Loan services contribute the highest transaction value.
📅 Weekday transaction activity significantly exceeds weekend usage.
⭐ High-value users can be targeted for premium offerings and loyalty programs.
